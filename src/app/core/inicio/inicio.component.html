Criterios de Aceptacion Trabajo Final

Patron Redux:

Aplicado en feature Cursos.

API:

Se implementan los metodos get, post, delete y put en los servicios:

  • alumnos.service.ts
  • cursos.service.ts
  • usuarios.service.ts
  • inscripciones.service.ts

Login:

core/login/loginModule.ts: Para loguearse se pueden usar los usuarios: admin 123456, user 123456

Guards:

core/auth.guard: Valida que este logueado un usuario para acceder a las rutas

core/perfil-admin.guard: Valida que el perfil de usuario sea "admin" para ingresar a la ruta /usuarios/

Lazzy Loading:

Se aplica loadChildren en todas las rutas.

Pruebas Unitarias:

Pruebas Unitarias Modulo: alumnos/componentes/alta/alta.component.spec.ts

Pruebas Unitarias servicio: alumnos/services/alumnos.service.spec.ts

Modulos

Alumnos: Se pueden realizar el ABM de alumnos.

Cursos: Se puede realizar el ABM de cursos.

Inscripciones: Se pueden realizar las asignaciones de cursos a los alumnos.
Al buscar un alumno se muestra en la tabla los cursos a los que se encuentra inscripto.
Se pueden eliminar las inscripciones del alumno.

Core: Contiene los servicios de login

Shared: Contiene el navbar y el sidebar. Contiene el modulo Material para importar todo lo relacionado con Angular Material.

Routing:

/alumnos/alta

/cursos/alta

/inscripciones/alta

Pipes Personalizados:

nombreApellido: Muestra el nombre y el apellido.

edad: Calcula y muestra la edad a partir de la fecha de nacimiento ingresada.

pais: Imprime la bandera correspondiente a la nacionalidad del alumno.

Directivas:

encabezados: Formatea en 15px los encabezados de la tabla.



Estructura realizada con Bootstrap.
Estilos de Angular Material y CSS
